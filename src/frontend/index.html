Como funciona o TF-IDF

O TF-IDF (Term Frequency – Inverse Document Frequency) atribui um peso a cada par (termo, documento). Termos frequentes num documento mas raros na coleção recebem pesos altos — são os mais discriminativos.

TF(t, d) = freq(t, d) / total_termos(d)
IDF(t) = log( N / df(t) ) + 1
TF-IDF(t, d) = TF(t, d) × IDF(t)

Similaridade = cosine( vec_query, vec_doc ) = (q · d) / (‖q‖ × ‖d‖)

Pesquisa Booleana com Índice Invertido

Na pesquisa booleana cada termo é resolvido como uma lista de doc_ids (posting list). Os operadores combinam essas listas:

machine AND learning→ interseção das postings de "machine" ∩ "learning"
neural OR deep→ união: postings("neural") ∪ postings("deep")
machine NOT neural→ diferença: postings("machine") \ postings("neural")
(neural OR deep) AND learning→ parênteses têm prioridade; NOT > AND > OR

A interseção é optimizada com skip pointers: ao percorrer as postings, ponteiros de salto (espaçados por √n) permitem saltar blocos, reduzindo comparações de O(n) para O(√n) no melhor caso.

Estrutura do Índice Invertido

Para cada termo único da coleção, o índice guarda a sua posting list: a lista de documentos onde aparece, com a frequência (TF) e as posições no texto.

learning→
doc_2 (tf=4)doc_5 (tf=2)doc_9 (tf=1)doc_12 (tf=3)
neural→
doc_1 (tf=6)doc_5 (tf=1)doc_7 (tf=2)
retrieval→
doc_3 (tf=5)doc_8 (tf=2)doc_11 (tf=1)

O índice suporta atualizações incrementais — novos documentos podem ser indexados sem reconstruir tudo — e persiste em JSON para recarregamento rápido.

Pré-processamento de Texto (NLP)

Antes de indexar ou pesquisar, o texto passa por um pipeline configurável:

"Running neural networks efficiently"
→ tokenização: ["Running","neural","networks","efficiently"]
→ lowercase: ["running","neural","networks","efficiently"]
→ stop words: ["running","neural","networks","efficiently"] (nenhuma removida)
→ stemming: ["run","neural","network","effici"]
→ lematização: ["run","neural","network","efficiently"]

Stemming (Porter) corta sufixos de forma agressiva e rápida — "studies" → "studi". Lematização (WordNet) é mais lenta mas produz raízes reais — "studies" → "study". Podes escolher o modo no painel de opções.
